# /deliberate — 博弈工作流 v2.0

> 触发方式：`/deliberate <议题>` 或 `/deliberate --on <exp-id>`
> 引擎配置：`.metap/engines/deliberation.yaml`
> 自治级别：辩论过程 L3 / 结论确认 L1
> 版本：v2.0（4 角色 3 轮完整版 + 质量守护强化）

## 角色定义

| 角色 | 图标 | 立场 | 核心职责 |
|------|------|------|---------|
| **倡导者** | 🟢 | 为提案辩护 | 阐述价值、引用证据、回应质疑 |
| **质疑者** | 🔴 | 寻找漏洞 | 识别风险、逻辑检查、边界条件 |
| **务实者** | 🟡 | 评估约束 | 资源分析、ROI 计算、MVP 路径 |
| **远见者** | 🔵 | 扩展视野 | 长期影响、跨领域启示、二阶效应 |

## 前置条件

- MetaP.md 已加载
- `.metap/engines/deliberation.yaml` 已加载
- 如果 `--on <exp-id>`，对应的探索报告已完成

## 工作流步骤

### Step 1: 议题确认与角色分配

1. 确认辩论议题
2. 加载 4 个辩论角色的 prompt 约束
3. 如果来自探索，提取核心发现和矛盾点作为辩论素材
4. 向用户确认："即将就 `{topic}` 发起辩论，参与角色：🟢🔴🟡🔵。开始？"

### Step 2: Round 1 — 陈述与质疑

**🟢 倡导者发言**（max 800 tokens）：
- 陈述核心价值主张，每个论点引用来源

**🔴 质疑者发言**（max 800 tokens）：
- 针对每个论点提出具体质疑
- 区分"致命缺陷 ❌"和"可管理的风险 ⚠️"

> **📋 质量守护 #1**: 检查双方是否坚守立场约束、是否有证据支撑

### Step 3: Round 2 — 回应、约束、视野

**🟢 倡导者回应**（max 600 tokens）：逐条回应质疑
**🟡 务实者评估**（max 800 tokens）：资源分析 + ROI + MVP 建议 + 具体工作量估算
**🔵 远见者补充**（max 600 tokens）：长远视角 + 跨学科启示 + 二阶效应

> **📋 质量守护 #2**: 务实者是否给了具体估算？远见者是否锚定在当前议题？

### Step 4: Round 3 — 立场修正与共识形成

每个角色独立表态（each max 400 tokens）：
- 🟢 坚持/让步/修正方案
- 🔴 已解决/仍存在/致命风险分类
- 🟡 推荐路径 + 工作量 + MVP vs Full
- 🔵 长期价值判断 + 二阶效应

**共识形成**：
- **共识**（3/4+ 同意）→ 标记为共识结论
- **多数意见**（2/4 同意）→ 标记为多数意见
- **分歧**（无多数）→ 结构化呈现各方观点

> **📋 质量守护 #3: 虚假共识检测** — 全票一致 → 🔴 强制 devil's advocate

### Step 5: 结论呈现 — ⚠️ 人类检查点

**必须等待用户审批**

### Step 6: 知识注册与记忆

1. 辩论涌现的新概念 → Concept（maturity: seed）
2. 验证的已有概念 → 提升 maturity
3. 确认的矛盾 → 建立 `contradicts` Link
4. Episodic memory → `.metap/memory/episodes/`
5. 辩论记录 → `.metap/explorations/{exp-id}/deliberation.md`

## 辩论质量守护总览

| 守护规则 | 检查时机 | 处理方式 |
|----------|---------|---------|
| **立场锁定** | 每轮发言后 | 偏离 stance → 下轮提示回归 |
| **证据要求** | Round 1-2 | 无证据论点标注 [unsubstantiated] |
| **具体估算** | Round 2 (🟡) | 务实者未给具体数字 → 要求补充 |
| **锚定检查** | Round 2 (🔵) | 远见者脱离议题 → 提示聚焦 |
| **虚假共识** | Round 3 | 全票一致 → 🔴 强制 devil's advocate |
| **长度控制** | 全程 | 超限 → 压缩后继续 |
