# Clean Code

## 核心指令

1. **命名即文档**：变量/函数名应自解释，不需要注释来解释"是什么"
2. **函数短小**：每个函数 < 30 行，只做一件事
3. **文件短小**：每个文件 < 300 行，超过则拆分
4. **消除重复**：DRY — 相同逻辑不出现两次
5. **注释说"为什么"而非"是什么"**：代码本身说明做什么，注释解释为什么
6. **错误处理**：不用返回码，用异常；不吞异常
7. **一致性**：同一项目内命名风格、缩进、导入顺序保持一致

## Gate 检查清单

- [ ] 是否有超过 30 行的方法？
- [ ] 是否有超过 300 行的文件？
- [ ] 是否有重复代码？
- [ ] 命名是否自解释？
