# Given-When-Then 验收标准

## 核心指令

用 GWT 三段式为每个 User Story 编写验收标准：

```
Given [前置条件/上下文]
When  [用户动作/事件触发]
Then  [期望结果/系统行为]
```

### 编写规范

| 元素 | 要求 | 反例 |
|------|------|------|
| **Given** | 明确的初始状态，可复现 | ❌ "系统正常" |
| **When** | 单一动作，主动语态 | ❌ "用户做了很多操作" |
| **Then** | 可验证的结果，有具体值 | ❌ "系统正确处理" |

### 扩展语法

| 关键词 | 用途 | 示例 |
|--------|------|------|
| `And` | 多个前置条件或结果 | `And 库存数量减少 5` |
| `But` | 排除/否定条件 | `But 不发送通知邮件` |

## AI 执行指引

1. 每个 Story 至少 1 个 GWT 场景，复杂 Story 需要 3-5 个
2. **必须覆盖的场景类型**：

| 场景类型 | 必须？ | 示例 |
|----------|--------|------|
| Happy Path | ✅ 必须 | 正常流程成功完成 |
| 边界条件 | ✅ 必须 | 数量为 0、最大值 |
| 权限不足 | 建议 | 非授权用户执行操作 |
| 异常输入 | 建议 | 空值、超长文本 |
| 并发场景 | 视情况 | 同时编辑同一记录 |

3. GWT 写完后，检查是否**可直接转化为自动化测试**——如果不能，说明 GWT 不够具体
4. 命名规则：`场景_{happy|edge|error}_{序号}`

## 示例

```yaml
scenarios:
  - name: "创建采购订单_happy_001"
    given: "用户登录为采购经理，且供应商 'ABC' 存在"
    when: "用户创建采购订单，选择供应商 'ABC'，添加产品 'X' 数量 100"
    then: "采购订单状态为 'draft'，总金额 = 100 × 单价"

  - name: "创建采购订单_edge_001"
    given: "用户登录为采购经理"
    when: "用户创建采购订单，不选择供应商"
    then: "系统显示验证错误：'供应商为必填项'"
```

## Gate 检查清单

- [ ] 每个 Story 是否至少有 1 个 GWT 场景？
- [ ] 是否覆盖了 Happy Path + 至少 1 个边界条件？
- [ ] Given 是否足够具体（可复现）？
- [ ] Then 是否可验证（有具体数值或状态）？
- [ ] 每个 GWT 是否可直接映射为 S6 测试用例？
